## 🚀 Quick Start

### Prerequisites

1. **Node.js** (LTS) - Download
2. **Python 3.12+** - Download
3. **Git** - Download
4. **VS Code** - Download

### Step 1: Configure Environment Variables

```powershell
# Copy environment template
Copy-Item .env.example .env
Copy-Item backend\.env.example backend\.env
Copy-Item frontend\.env.local.example frontend\.env.local
```

Then edit each `.env` file with your actual credentials.

### Step 2: Set Up Supabase (The Vault)

1. Create account at supabase.com
2. Create a new project
3. Go to **Project Settings > API** and copy:
   - Project URL
   - Anon (public) key
   - Service role key
4. Go to **SQL Editor** and run the schema from `database/rbac-schema.sql`

### Step 3: Start the Backend (The Brain)

```powershell
cd backend

# Create virtual environment
python -m venv venv

# Activate it
.\venv\Scripts\Activate.ps1

# Install dependencies
pip install -r requirements.txt

# Run the server
python main.py
```

**Production:** `[URL_REMOVED] ✅

### Step 4: Start the Frontend (The Storefront)

```powershell
cd frontend

# Install dependencies
npm install

# Run development server
npm run dev
```

The website will be running at `[URL_REMOVED]

## 🔐 RBAC (Role-Based Access Control)

The system supports 4 roles with hierarchical permissions:

| Role | Level | Permissions |
|------|-------|-------------|
| **Owner** | 4 | Full access - can manage users, see revenue, delete data |
| **Admin** | 3 | Manage content, users, and operations |
| **Staff** | 2 | View and edit assigned data |
| **Viewer** | 1 | Read-only access |

### How to Assign Roles

1. After a user signs up, get their user ID from Supabase
2. In Supabase SQL Editor, run:

```sql
INSERT INTO user_roles (user_id, role, org_id) 
VALUES ('user-uuid-here', 'staff', 'org-uuid-here');
```

## 🆘 Troubleshooting

### Backend won't start
- Ensure Python 3.12+ is installed: `python --version`
- Check virtual environment is activated
- Verify all packages installed: `pip list`

### Frontend won't start
- Ensure Node.js is installed: `node --version`
- Delete `node_modules` and run `npm install` again
- Check `.env.local` has all required variables

### Supabase connection issues
- Verify URL format: `[URL_REMOVED]
- Ensure you're using the correct anon key (not service key in frontend)
- Check Row Level Security policies are enabled
